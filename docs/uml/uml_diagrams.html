2. Đặc tả chức năng chi tiết

2.1 Đặc tả chức năng Nhập kho

Mô tả:

Chức năng này cho phép Nhân viên cập nhật số lượng và giá nhập kho của sản phẩm vào hệ thống khi có hàng mới từ nhà cung cấp thông qua Phiếu nhập kho.

Actor: Nhân viên

Điều kiện trước:

  • Nhân viên đã đăng nhập vào hệ thống với quyền hạn tương ứng
  • Hệ thống đã có danh mục Nhà cung cấp và Danh mục sản phẩm

Tình huống chính - Nhập kho thành công:

  1. Nhân viên truy cập vào chức năng Quản lý kho
  2. Hệ thống hiển thị danh sách các phiếu nhập kho hiện có
  3. Nhân viên chọn chức năng Tạo phiếu nhập mới
  4. Hệ thống hiển thị biểu mẫu nhập thông tin phiếu nhập
  5. Nhân viên điền thông tin: Nhà cung cấp, Ngày nhập, Danh sách sản phẩm (mã, tên, số lượng, đơn giá), Ghi chú
  6. Hệ thống tự động tính toán tổng giá trị phiếu nhập
  7. Nhân viên kiểm tra và nhấn nút Xác nhận/Lưu
  8. Hệ thống kiểm tra tính hợp lệ của dữ liệu
  9. Hệ thống ghi nhận thông tin vào cơ sở dữ liệu
  10. Hệ thống cập nhật số lượng tồn kho
  11. Hệ thống hiển thị thông báo thành công

Tình huống thay thế a - Dữ liệu không hợp lệ:

  1. Hệ thống phát hiện Số lượng ≤ 0 hoặc Đơn giá ≤ 0
  2. Hệ thống hiển thị thông báo lỗi tại các ô tương ứng
  3. Quay lại bước 5

Tình huống thay thế b - Hủy nhập kho:

  1. Nhân viên chọn chức năng Hủy
  2. Hệ thống cảnh báo dữ liệu chưa được lưu
  3. Nhân viên xác nhận hủy
  4. Hệ thống quay về giao diện danh sách phiếu nhập

2.2 Đặc tả chức năng Sửa sản phẩm

Mô tả:

Chức năng cho phép Admin cập nhật thông tin sản phẩm: tên, loại, giá bán, số lượng, chất liệu, kích thước, hình ảnh và mô tả.

Actor: Admin

Điều kiện trước:

  • Người dùng có quyền chỉnh sửa sản phẩm
  • Sản phẩm cần sửa tồn tại trên hệ thống

Tình huống chính - Sửa sản phẩm thành công:

  1. Admin truy cập vào chức năng Quản lý sản phẩm
  2. Hệ thống hiển thị danh sách các sản phẩm
  3. Admin nhấn nút Chỉnh sửa sản phẩm
  4. Hệ thống hiển thị thông tin chi tiết cần chỉnh sửa
  5. Admin nhập các thông tin cần chỉnh sửa
  6. Hệ thống kiểm tra tính hợp lệ của dữ liệu
  7. Admin nhấn nút Lưu
  8. Hệ thống hiển thị thông báo thành công và lưu vào database

2.3 Đặc tả chức năng Đặt hàng

Mô tả:

Chức năng cho phép Khách hàng tạo đơn hàng từ các sản phẩm đã chọn trong giỏ hàng.

Actor: Khách hàng

Điều kiện trước:

  • Khách hàng đã đăng nhập thành công
  • Giỏ hàng có ít nhất 1 sản phẩm hợp lệ

Tình huống chính - Đặt hàng thành công:

  1. Hệ thống hiển thị giao diện giỏ hàng
  2. Hệ thống tự động kiểm tra tồn kho cho tất cả sản phẩm
  3. Khách hàng chọn các sản phẩm cần mua và nhấn Đặt hàng
  4. Hệ thống hiển thị form nhập thông tin giao hàng
  5. Khách hàng nhập/xác nhận thông tin giao hàng
  6. Hệ thống tính toán phí vận chuyển
  7. Hệ thống hiển thị bảng tóm tắt đơn hàng
  8. Khách hàng kiểm tra và nhấn Xác nhận đặt hàng
  9. Hệ thống ghi nhận đơn hàng và cập nhật tồn kho
  10. Hệ thống hiển thị thông báo thành công kèm mã đơn

2.4 Đặc tả chức năng Thanh toán đơn hàng

Mô tả:

Chức năng cho phép Khách hàng thực hiện thanh toán đơn hàng qua nhiều phương thức (QR PayOS, COD).

Actor: Khách hàng

Điều kiện trước:

  • Khách hàng đã đăng nhập
  • Đơn hàng đã được tạo
  • Hệ thống thanh toán hoạt động bình thường

Tình huống chính - Thanh toán QR thành công:

  1. Khách hàng nhấn nút Thanh toán ở đơn hàng
  2. Hệ thống hiển thị thông tin đơn hàng và phương thức
  3. Khách hàng chọn phương thức QR PayOS
  4. Khách hàng xác nhận thông tin và nhấn Xác nhận
  5. Hệ thống tạo mã QR thông qua PayOS
  6. Khách hàng quét mã bằng app ngân hàng
  7. Hệ thống kiểm tra và xác nhận giao dịch
  8. Hệ thống chuyển trạng thái thành "Đã thanh toán"

Tình huống thay thế - COD:

  1. Khách hàng chọn COD
  2. Hệ thống ghi nhận COD
  3. Trạng thái chuyển sang "Chờ xác nhận"

Tình huống thay thế - Hủy thanh toán:

  1. Khách hàng nhấn Hủy
  2. Hệ thống giữ trạng thái "Chưa thanh toán"
  3. Quay về màn hình chính

8. Tạo mã nguồn từ thiết kế (Design to Code)

Nguyên tắc Design to Code:
  • Biến toàn cục của lớp (class-level variables)
  • Biến cục bộ của hàm (local variables)
  • Tham số của hàm (method parameters)
  • 1 lớp trong thiết kế → 1 class trong code

8.1 Lớp OrderService (Hình 7.1)

Design → Code: OrderService.java

@Service
@RequiredArgsConstructor
@Slf4j
public class OrderService {
    
    // === Biến toàn cục của lớp (Class-level variables) ===
    private final OrderRepository orderRepository;
    private final OrderItemRepository orderItemRepository;
    private final WarehouseService warehouseService;
    private final CartService cartService;
    
    // === Các phương thức (Methods) ===
    
    // M1: xacNhanDatHang() - Tham khảo Hình 6.1
    @Transactional
    public OrderDTO xacNhanDatHang(Long userId, ShippingInfo shippingInfo) {
        // Biến cục bộ (Local variables)
        List<CartItemDTO> cartItems = cartService.getCartItems(userId);
        List<OrderItem> orderItems = new ArrayList<>();
        BigDecimal subtotal = BigDecimal.ZERO;
        
        // Kiểm tra tồn kho
        for (CartItemDTO item : cartItems) {
            StockStatus stock = warehouseService.checkStock(
                item.getProductId(),  // Tham số
                item.getQuantity()     // Tham số
            );
            
            if (stock.isOutOfStock()) {
                throw new RuntimeException(
                    "Sản phẩm " + item.getProductName() + " đã hết hàng"
                );
            }
            
            // Tạo OrderItem
            OrderItem orderItem = OrderItem.builder()
                .productName(item.getProductName())
                .price(item.getPrice())
                .quantity(item.getQuantity())
                .total(item.getPrice().multiply(BigDecimal.valueOf(item.getQuantity())))
                .build();
            
            orderItems.add(orderItem);
            subtotal = subtotal.add(orderItem.getTotal());
        }
        
        // Tính tổng
        BigDecimal totalAmount = calculateTotal(subtotal, shippingInfo);
        
        // Tạo Order
        Order order = Order.builder()
            .orderNumber(generateOrderNumber())
            .user(userService.getUserById(userId))
            .subtotal(subtotal)
            .shippingFee(shippingInfo.getShippingFee())
            .totalAmount(totalAmount)
            .status(OrderStatus.PENDING)
            .paymentStatus(PaymentStatus.PENDING)
            .shippingAddress(shippingInfo.getFullAddress())
            .shippingPhone(shippingInfo.getPhone())
            .build();
        
        // Lưu Order
        order = orderRepository.save(order);
        
        // Trừ tồn kho
        for (OrderItem item : orderItems) {
            warehouseService.reduceStock(item.getProductId(), item.getQuantity());
        }
        
        // Xóa giỏ hàng
        cartService.clearCart(userId);
        
        return toDTO(order);
    }
    
    // M2: huyDatHang() - Tham khảo Hình 6.2
    @Transactional
    public void huyDatHang(Long orderId, String reason) {
        // Biến cục bộ
        Order order = orderRepository.findById(orderId)
            .orElseThrow(() -> new RuntimeException("Không tìm thấy đơn hàng"));
        
        // Kiểm tra có thể hủy không
        if (order.getStatus() != OrderStatus.PENDING) {
            throw new RuntimeException("Không thể hủy đơn hàng đã xác nhận");
        }
        
        // Hoàn lại tồn kho
        for (OrderItem item : order.getItems()) {
            warehouseService.restoreStock(
                item.getProduct().getId(),  // Tham số
                item.getQuantity()           // Tham số
            );
        }
        
        // Cập nhật trạng thái
        order.setStatus(OrderStatus.CANCELLED);
        order.setCancellationReason(reason);
        order.setCancelledAt(LocalDateTime.now());
        
        orderRepository.save(order);
    }
    
    // Hàm private
    private BigDecimal calculateTotal(BigDecimal subtotal, ShippingInfo shipping) {
        return subtotal
            .add(shipping.getShippingFee() != null ? shipping.getShippingFee() : BigDecimal.ZERO)
            .subtract(shipping.getDiscountAmount() != null ? shipping.getDiscountAmount() : BigDecimal.ZERO);
    }
}

8.2 Lớp WarehouseService (Hình 7.6)

Design → Code: WarehouseService.java

@Service
@RequiredArgsConstructor
@Slf4j
public class WarehouseService {
    
    // === Biến toàn cục của lớp ===
    private final WarehouseRepository warehouseRepository;
    private final WarehouseProductRepository warehouseProductRepository;
    private final WarehouseTransactionRepository transactionRepository;
    
    // === Các phương thức (Methods) ===
    
    // M1: kiemTraTonKho() - Tham khảo Hình 6.1
    public StockStatus kiemTraTonKho(Long productId, Integer requestedQty) {
        // Biến cục bộ
        Integer availableStock = warehouseProductRepository
            .getTotalStockByProductId(productId);
        
        if (availableStock == null) {
            availableStock = 0;
        }
        
        if (availableStock >= requestedQty) {
            return StockStatus.builder()
                .productId(productId)
                .available(true)
                .availableQuantity(availableStock)
                .requestedQuantity(requestedQty)
                .build();
        } else {
            return StockStatus.builder()
                .productId(productId)
                .available(false)
                .availableQuantity(availableStock)
                .requestedQuantity(requestedQty)
                .message("Chỉ còn " + availableStock + " sản phẩm")
                .build();
        }
    }
    
    // M2: truTonKho() - Tham khảo Hình 6.1
    @Transactional
    public void truTonKho(Long productId, Integer quantity) {
        // Biến cục bộ
        List<WarehouseProduct> products = warehouseProductRepository
            .findByProductIdOrderByImportDateAsc(productId);
        
        int remaining = quantity;
        for (WarehouseProduct wp : products) {
            if (remaining <= 0) break;
            
            int deduct = Math.min(wp.getQuantity(), remaining);
            wp.setQuantity(wp.getQuantity() - deduct);
            remaining -= deduct;
            
            warehouseProductRepository.save(wp);
            
            // Tạo transaction
            createTransaction(
                wp.getWarehouse().getId(),  // Tham số
                productId,                    // Tham số
                deduct,                       // Tham số
                wp.getImportPrice()          // Tham số
            );
        }
    }
    
    // M3: capNhatTonKho() - Tham khảo Hình 7.2
    @Transactional
    public void capNhatTonKho(Long productId, Integer quantity, BigDecimal price) {
        // Biến cục bộ
        Warehouse warehouse = warehouseRepository
            .findFirstByStatus(WarehouseStatus.ACTIVE)
            .orElseThrow(() -> new RuntimeException("Không có kho hoạt động"));
        
        // Tìm hoặc tạo mới
        WarehouseProduct wp = warehouseProductRepository
            .findByWarehouseIdAndProductId(warehouse.getId(), productId)
            .orElse(WarehouseProduct.builder()
                .warehouse(warehouse)
                .product(productRepository.findById(productId).get())
                .quantity(0)
                .build());
        
        // Cập nhật số lượng và giá
        wp.setQuantity(wp.getQuantity() + quantity);
        wp.setImportPrice(price);
        wp.setLastImportDate(LocalDateTime.now());
        
        warehouseProductRepository.save(wp);
        
        // Tạo transaction ghi nhận nhập kho
        createTransaction(
            warehouse.getId(),   // Tham số
            productId,           // Tham số
            quantity,            // Tham số
            price                // Tham số
        );
    }
    
    // Hàm private
    private void createTransaction(Long warehouseId, Long productId, 
                                   Integer quantity, BigDecimal price) {
        WarehouseTransaction transaction = WarehouseTransaction.builder()
            .transactionCode("IMP-" + System.currentTimeMillis())
            .transactionType(TransactionType.IMPORT)
            .warehouse(warehouseRepository.findById(warehouseId).get())
            .product(productRepository.findById(productId).get())
            .quantity(quantity)
            .unitPrice(price)
            .status(TransactionStatus.COMPLETED)
            .build();
        
        transactionRepository.save(transaction);
    }
}

8.3 Lớp PaymentService (Hình 7.4)

Design → Code: PaymentService.java

@Service
@RequiredArgsConstructor
@Slf4j
public class PaymentService {
    
    // === Biến toàn cục của lớp ===
    private final OrderRepository orderRepository;
    private final PayOSService payOSService;
    
    // === Các phương thức (Methods) ===
    
    // M1: thanhToanDonHang() - Tham khảo Hình 7.4
    @Transactional
    public Map<String, Object> thanhToanDonHang(Long orderId, String paymentMethod) {
        // Biến cục bộ
        Order order = orderRepository.findById(orderId)
            .orElseThrow(() -> new RuntimeException("Không tìm thấy đơn hàng"));
        
        // Kiểm tra trạng thái
        if (order.getPaymentStatus() == PaymentStatus.PAID) {
            throw new RuntimeException("Đơn hàng đã được thanh toán");
        }
        
        if (order.getStatus() == OrderStatus.CANCELLED) {
            throw new RuntimeException("Đơn hàng đã bị hủy");
        }
        
        Map<String, Object> result = new HashMap<>();
        result.put("orderId", order.getId());
        result.put("orderNumber", order.getOrderNumber());
        
        // Xử lý theo phương thức
        if ("PAYTOS".equalsIgnoreCase(paymentMethod) || "QR".equalsIgnoreCase(paymentMethod)) {
            return initiateQrPayment(order);
        } else if ("COD".equalsIgnoreCase(paymentMethod)) {
            return initiateCodPayment(order);
        } else {
            throw new RuntimeException("Phương thức thanh toán không hợp lệ");
        }
    }
    
    // M2: huyThanhToanDonHang() - Tham khảo Hình 7.5
    @Transactional
    public void huyThanhToanDonHang(Long orderId) {
        // Biến cục bộ
        Order order = orderRepository.findById(orderId)
            .orElseThrow(() -> new RuntimeException("Không tìm thấy đơn hàng"));
        
        // Chỉ cho phép hủy khi đơn còn PENDING
        if (order.getStatus() != OrderStatus.PENDING) {
            throw new RuntimeException("Chỉ có thể hủy đơn hàng đang chờ xác nhận");
        }
        
        // Xóa thông tin PayOS nếu có
        if (order.getPayosCheckoutUrl() != null || order.getPayosOrderCode() != null) {
            order.setPayosCheckoutUrl(null);
            order.setPayosOrderCode(null);
            orderRepository.save(order);
        }
        
        // Giữ nguyên payment_status = PENDING
        log.info("Đã hủy thanh toán cho đơn hàng: " + order.getOrderNumber());
    }
    
    // Hàm private
    private Map<String, Object> initiateQrPayment(Order order) {
        order.setPaymentMethod(Order.PaymentMethod.PAYTOS);
        orderRepository.save(order);
        
        try {
            // Tạo payment link từ PayOS
            PayOSService.PaymentLinkResult linkResult = payOSService.createPaymentLinkWithId(
                order.getId(),              // Tham số
                order.getOrderNumber(),     // Tham số
                order.getTotalAmount()     // Tham số
            );
            
            // Lưu thông tin
            order.setPayosCheckoutUrl(linkResult.checkoutUrl());
            order.setPayosOrderCode(String.valueOf(linkResult.orderCode()));
            orderRepository.save(order);
            
            Map<String, Object> result = new HashMap<>();
            result.put("checkoutUrl", linkResult.checkoutUrl());
            result.put("expiresAt", linkResult.expiredAt());
            result.put("message", "Vui lòng thanh toán trước " + linkResult.expiredAt());
            
            return result;
        } catch (Exception e) {
            log.error("PayOS error: {}", e.getMessage());
            throw new RuntimeException("Không thể tạo liên kết thanh toán");
        }
    }
    
    private Map<String, Object> initiateCodPayment(Order order) {
        order.setPaymentMethod(Order.PaymentMethod.COD);
        orderRepository.save(order);
        
        Map<String, Object> result = new HashMap<>();
        result.put("message", "Thanh toán khi nhận hàng");
        result.put("paymentMethod", "COD");
        
        return result;
    }
}
